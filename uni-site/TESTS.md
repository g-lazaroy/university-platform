# Σχέδιο Δοκιμών - Εργασία Μέρος 2

## Στόχος
Επαλήθευση της σωστής λειτουργίας του dashboard, RBAC, υποβολής εργασιών, βαθμολόγησης και συγχρονισμού δεδομένων.

## Περιβάλλον Δοκιμών
- XAMPP (Apache + MySQL)
- Browser: Chrome/Firefox
- Responsive: Desktop, Tablet, Mobile (Bootstrap 5)

## 1. Functional Testing

### Σενάριο 1: Υποβολή & Βαθμολόγηση Εργασίας (Συγχρονισμός)
1. Login ως φοιτητής (`nikos@uni.gr` / `123456`)
2. "Εργασίες" → Επίλεξε εργασία → Upload αρχείο
   → Μήνυμα "Υποβλήθηκε!"
3. Αποσύνδεση
4. Login ως καθηγητής (`papadopoulos@uni.gr` / `prof123`)
5. "Υποβολές" → Βλέπει την υποβολή → Βαθμολογεί (π.χ. 7.5 + σχόλιο)
6. Αποσύνδεση + Login ως φοιτητής
7. "Βαθμοί" → Βλέπει βαθμό + σχόλιο
   → **Συγχρονισμός επιτυχής**


### Σενάριο 2: Δημιουργία Μαθήματος & Εργασίας
1. Login ως καθηγητής
2. "Μαθήματα" → Δημιουργία νέου μαθήματος
3. "Νέα Εργασία" → Ανάρτηση
4. Login ως φοιτητής → "Εργασίες"
   → Βλέπει τις εργασίες

## 2. Responsive Testing
- Δοκιμή σε:
  - Desktop (1920px)
  - Tablet (768px)
  - Mobile (375px)
- **Αναμενόμενο:**
  - Navbar → hamburger menu σε mobile
  - Κάρτες/πίνακες → προσαρμόζονται
  - Forms → ευανάγνωστα

## 3. Performance & Stability
- Page load < 1.5s
- Upload αρχείων <10MB → επιτυχής
- Πολλαπλές συνδέσεις → sessions OK

## Συμπέρασμα
Η εφαρμογή λειτουργεί σταθερά, είναι responsive και καλύπτει όλες τις απαιτήσεις.
